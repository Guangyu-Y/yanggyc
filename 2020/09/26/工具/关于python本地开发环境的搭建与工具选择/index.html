方案

方案1：本地开发

如果仅需要做本地的简单脚本开发，比较好用的还是vscode。可以直接创建脚本，编辑运行即可，不需要像pycharm那样占用过多的系统内存。

如果是工程类型的项目，外部包依赖较多的情况下，如果项目的运行环境是windows，则可以在本地使用pycharm开发，因为pycharm在代码调试、依赖包源码查看等开发体验上有比较不错的表现，可能唯一的缺点就是打开后占用系统资源比较高；如果项目运行环境是linux系统，也可在本地进行开发，但是开发环境和运行环境是异构的，可能会导致有些开源依赖包的使用方式在windows和linux有所不同，进而导致在联调过程中进行反工。因此当项目的运行环境是linux时，建议采用远程开发。见方案二、三。

方案2：远程开发

  1. VSCode远程开发。当ssh连接远程服务器不需要跳板机时，可直接采用vscode，配置远程目录，进行远程开发。开发过程中可以实时保存至远程服务器代码目录，并进行运行，但是比较依赖网络的稳定性，网络不稳定的情况下，可能会在保存的过程中卡住。另一个挑战就是代码版本控制方面，需要使用GIT命令做相应操作，故需要开发任务有比较扎实的git知识。也可使用vscode的git插件。
  2. VIM开发。个人建议这种方式只开发简单脚本，复杂的工程开发，还是有一定的学习成本。
  3. Pycharm使用远程Python运行项目。在windows本地做代码开发，在pycharm中配置python解释器的时候，配置linux服务器上的python环境，可做到本地开发，使用远程环境运行的效果。但是如果你登录远程环境得通过跳板机，这种方式就比较复杂了，需要配置网络代理。不建议这么做，因为可能会被公司安全部谈话。此时建议在windows安装wsl，使用wsl的python配置解释器。
  4. Pycharm同步远程代码。建议在不需要跳板机连接的linux机器上或者本地wsl中开发代码，pycharm tools deployment中配置远程目录，设置本地代码与远程代码的同步目录，可选择使用sftp、rsync进行两个路径的代码同步，也可以直接打开远程的代码进行编辑、保存。如果本地内存不够的情况下，可直接使用vim做简单开发，集成调试时，使用pycharm。
