数据库报错整理

Posted on 2021-09-26 Edited on 2022-12-27 In 数据库

1.mysql数据库在连接时报

  1. 先查看 /etc/rc.d/init.d/mysqld status 看看m y s q l 是否已经启动.
  2. 确定你的mysql.sock是不是在那个位置，
    mysql -u 你的mysql用户名 -p -S /var/lib/mysql/mysql.sock
  3. 试试：service mysqld start
  4. 如果是权限问题，则先改变权限 #chown -R mysql:mysql /var/lib/mysql
  5. 在确认文件存在的情况下，备份文件，重启mysql服务。

2.针对mysql数据库忘记root用户的密码

  1. 配置忽略密码的选项，路径：/etc/my.cnf skip-grant-tables

3.mysql 在my.cnf中的部分配置

  1. character_set_server=utf8 #服务端字符集设置
  2. init_connect=’SET NAMES utf8’
  3. lower_case_table_names=1 #忽略大小写
  4. max_connections=5000 #数据库最大连接数配置
  5. max_allowed_packet = 100M限制server接受的数据包的大小

4.数据类型

MySQL的四种BLOB类型
类型 大小(单位：字节)
TinyBlob 最大 255
Blob 最大 65K
MediumBlob 最大 16M
LongBlob 最大 4G

5.初始化修改密码

  • 初始化密码路径：/var/log/mysql.log

6.在修改了my.cnf之后，错误日志为无权限打开文件。

  1. 查看selinux状态；

loadrunner脚本函数整理

Posted on 2021-09-26 Edited on 2022-12-27 In 测试
  • lr_save_string(“A”,”B”); //将lr中定义的参数B保存为A。
  • lr_start_transaction(“transaction_name”); //事务开始
    • transaction_name:事务名
  • lr_end_transaction(“transaction_name”,status); //事务结束
    • transaction_name:事务名
    • status:事务的结束状态{LR_PASS(通过)、LR_FAIL(失败)、LR_AUTO（自动）、 LR_STOP(暂停)}
  • lr_set_transaction_status(Status);//事务结束前设置事务状态
  • long id = lr_start_transaction_instance（const char * transaction_name，long handle）;//标记事务实例的开始。事务实例是具有transaction_name的事务的发生。实例通过其句柄来识别，其将它们与同一事务的其他实例区分开。
    • transaction_name：事务的名称。
    • handle：父事务的句柄。值为0指示VuGen创建新的父事务。
  • lr_end_transaction_instance（id，LR_PASS）;//
  • lr_rendezvous(“集合点名”);//设置集合点
  • web_url
  • web_custom_request();//函数允许使用任何方法或主体创建自定义的HTTP请求，其使用范围更广。其中，Body中，如果是json类型的，引号需要使用 \ 进行转义。其他类型的，直接跟=后面即可。
  • web_image
  • web_add_cookie
  • web_add_header
  • web_reg_find(“Search=Body”,”Text=user_id”,”SaveCount=userId”,LAST);//验证响应是否符合期望，如通过搜索文本字符串
    • Search : 搜索字符串的搜索范围，如ALL、Body、Header
    • Text : 搜索的字符串
    • SaveCount : 匹配的数量，并存储在一个参数中
  • web_reg_save_param(“user_token”,”LB="user_token":"“,”RB="“,”Ord=1”,”Notfound=empty”,”Search=Body”,LAST);//
    • LB 参数的左边界
    • RB 参数的右边界
    • Ord 匹配的序号位置，默认为1
    • Notfound 当没有找到边界时的处理选项，选项有：Notfound=error - Notfound=empty
    • Search : 搜索字符串的搜索范围，如ALL、Body、Header
